# KCode AI 编码工作台 — 终版产品方案

> 存档日期：2026-05-29
> 状态：方案已定稿，待开发（参见 TASKS.md Phase 29）

---

## 一、方案总定位

KCode 采用 **一套内核、两套 UI 范式** 的双模式架构，彻底解决传统 AI 编码「聊天混乱、无流程、无管控、难复盘、用户输入繁重」的行业痛点。

- **底层**：统一单 Session 任务内核（上下文、终端、回溯、恢复、自检逻辑完全复用，无重构、无推翻）
- **上层**：支持**对话模式（传统兼容）/ 三张卡片工作台模式（工程化主推）** 一键切换
- **核心范式**：AI 全权执行、人类只做决策审批、最小化用户输入、结构化工程闭环

区别于所有竞品（Cursor / Claude Code）：

> 竞品是「聊天驱动编码」，本方案是 **「任务流程驱动 AI 编码」**，对齐工程师原生研发工作流，可控、可沉淀、可回溯、可企业化落地。

---

## 二、核心顶层设计（不变底层内核）

所有底层能力完全保留，双模式通用：

- 单任务独立 Session，完整上下文不中断
- 终端命令执行、日志归集、编译构建、自动化自检
- 任务阶段回溯、版本快照、异常恢复、断点续跑
- 文件变更 Diff、代码自检、自动排错、重试机制

**UI 只是视图层切换，业务内核 100% 复用**，无额外开发成本、无技术风险。

---

## 三、双模式架构（用户可配置切换）

### 模式 1：传统对话模式（兼容兜底）

- **适用场景**：微型改动、单行修改、临时调试、查日志、跑单命令
- **形态**：纯聊天流式交互，无卡片、无强制流程、快速轻量
- **固定绑定**：【小助手模块】，全程自由对话

### 模式 2：三张卡片任务工作台模式（主推高阶工程模式）

- **适用场景**：正式功能开发、模块迭代、需求改造、需要沉淀复盘的完整任务
- **形态**：阶段结构化卡片 + 版本迭代 + 楼中楼评论 + 异步通知 + 点点点审批
- **核心优势**：成果与过程分离、强流程约束、零冗余打字、全链路可控

---

## 四、三张核心卡片 — 终版固定流程（唯一标准研发闭环）

整套流程严格区分 AI 权责 / 用户权责，彻底解决「用户看不懂代码、不愿审代码、UI 测试无法自动化」的核心难点。

### 卡片 1：目标 & 方案卡片（任务启动 · 事前审批）

**生成时机**
用户提交正式任务需求后，AI 首轮自动生成 V1.0 版本。

**固定展示内容（无任何代码、用户零代码阅读）**
- 任务核心目标（人话对齐需求）
- 整体实现方案（逻辑步骤，自然语言）
- 精准改动文件清单
- 改动范围、风险等级、边界约束（明确「改什么、绝不改什么」）
- 可选快捷微调勾选项（结构化选项，无需打字）

**权责分工**
- AI：全权产出结构化方案与范围
- 用户：只做方向审批，不看代码、不读源码
- 操作仅三类点击：同意方案 / 驳回重改 / 勾选微调
- 支持卡片版本迭代：V1.0 → V1.1 … 确认后锁定方案，禁止乱改方向

**评论区能力**
卡片专属楼中楼对话，仅用于方案微调批注，沟通绑定当前版本，不污染全局上下文。

---

### 卡片 2：执行 & 自检卡片（事中执行 · AI 全责）

**生成时机**
方案锁定确认后，AI 自动进入执行流程，执行 + 自检完成后自动生成。

**固定展示内容**
- 任务执行状态、进度摘要
- 自动化执行结果（编译 / 构建 / 测试）
- AI 全维度自检报告（语法、逻辑、边界、报错修复）
- 终端完整日志（默认折叠，无需用户查看）

**权责分工（核心定规）**
- AI 全权负责所有执行层工作：改代码、跑命令、构建编译、自动化测试、日志分析、报错修复、重试调试
- 用户全程零参与、零操作、零查看
- **此阶段禁止用户介入**，彻底解放人工脏活

---

### 卡片 3：变更验收卡片（事后闭环 · 用户终审）

**生成时机**
AI 执行 + 自检 100% 完成后，自动推送验收任务，触发用户通知。

**固定展示内容**
- 本次所有文件变更汇总
- 功能实现结果摘要（人话）
- 自动化测试通过证明
- 人工验证指引（UI 交互 / 命令行核验入口）

**权责分工（解决 UI 测试、人工核验痛点）**
- AI 完成所有代码层、命令层、自动化层验证
- 唯一需要用户做的人工工作（仅在验收阶段）：
  - 按需查看终端最终输出
  - 手动点击 UI、验证交互、肉眼核验功能手感
- 用户终审操作：一键通过 / 驳回修改（支持极简文字批注 / 预设理由勾选）

**核心规则**
所有人工验证、UI 测试、终端复核，统一收口在验收阶段，执行自检阶段纯 AI 自治。

| 场景 | 模式 | 特点 |

| 中途升级 | 对话→任务 | 一键转为正式任务，自动生成三张卡片 |

## 六、终端执行架构（终版技术选型）

采用 **底层原生能力 + 上层统一调度** 双层架构，不重复造轮子、且支持多设备多协议扩展：

- **底层**：复用系统原生 Bash / PowerShell / VSCode 内置终端，保证兼容性、稳定性、环境一致性
- **上层**：自研统一终端调度工具
  - 能力：命令归集、任务绑定、版本关联、日志结构化存储
  - 扩展：支持本地、SSH 远程、Docker、嵌入式多设备协议
- **流程绑定**：所有执行日志自动汇入卡片 2、卡片 3，实现任务级完整回溯

| 壁垒 | 说明 |
|------|------|
| **范式壁垒** | 告别聊天式乱开发，落地「人定决策、AI 做执行」的工程化新范式 |
| **体验壁垒** | 用户极致低输入，95% 操作纯点击，无需打字、无需读代码 |
| **流程壁垒** | 前置方案审批，从根源减少 AI 返工、方向跑偏 |
| **沉淀壁垒** | 版本化卡片 + 结构化日志 + 任务全回溯，支持团队复盘与知识沉淀 |
| **适配壁垒** | 完美适配跨栈开发者（不懂底层源码、仅需验收功能）的真实协作习惯 |

## 九、总结

- **无底层重构**，开发成本极低，稳定性 100% 可控
- 双模式兼容轻重场景，覆盖所有 AI 编码开发需求
- 权责绝对清晰：AI 包干执行脏活，用户只做核心决策与最终 UI 验收
- 彻底解决行业痛点：聊天混乱、无流程、难管控、用户看不懂代码、UI 测试无法自动化
- 可直接进入开发落地，具备独立产品差异化与商业化价值

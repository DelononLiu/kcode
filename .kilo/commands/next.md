## 数据采集（grep 定位，不读全文）

```grep TASKS.md "^| P[0-9]"          // 提取表格行（~50行）
grep TASKS.md "^## Phase"           // Phase 标题
grep TASKS.md "^\*\*状态\*\*"       // 详情区块状态行
grep TASKS.md "^\*\*Phase\b"         // Phase 详情区块标题（含 Timeline 总览表）
```

未完成任务（`⬜`/`📋`）对应的详情区块再 read 其 `**涉及文件**` 行。
**不读 PROJECT.md / TASKS.md 全文** — 只按上面 grep 定位的行读。

## 逻辑

1. 表格行 → 按 Phase 分组，记状态/优先级
2. 对未完成任务，对比表行 vs 详情区块状态
3. 识别未完成任务间的依赖
4. 决策：

```
有未完成任务 → P0 未开始+空涉 → 调研
               P0 未开始+有涉 → 实现
               P1/P2 未开始 → 推荐无依赖的先推
               📋 已调研 → 推荐实现
全部完成 → 挑 1-2 产品方向推进
```

## 原则

- 只读表格行 + 未完成任务详情区块，不读已完成内容
- 不读 git 信息
- 输出结论 + 一句理由，不输出模板示例
